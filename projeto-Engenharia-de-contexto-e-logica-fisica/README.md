## 📋 Sobre o Projeto
Este repositório consolida projetos desenvolvidos para a disciplina de **Análise e Desenvolvimento de Sistemas**, explorando a tradução de cenários complexos do mundo real em algoritmos computacionais[cite: 13, 14]. O sistema abrange desde a gestão técnica de estoques e logística de transporte até simulações de navegação espacial em ambientes com obstáculos físicos[cite: 9, 14].

O foco central é a aplicação de lógica condicional para resolver problemas práticos, como o controle de troco monetário, o gerenciamento de reposição de produtos e a segurança em rotas de evacuação[cite: 7, 10, 14].

## 🚀 Funcionalidades Principais

### 📦 Gestão de Inventário e Comercial
*   **Controle de Saldos**: Sistema de entrada e saída de mercadorias com atualização dinâmica de estoque[cite: 9, 11].
*   **Validação de Transação**: Módulo que garante que o valor pago cubra a venda e calcula o troco otimizado por cédulas[cite: 7].
*   **Alertas de Insuficiência**: Notificações automáticas quando o saldo atual não atende à demanda[cite: 8, 10].

### 🚛 Logística de Distribuição
*   **Roteirização**: Cálculo de rotas baseado na distância e monitoramento do status de transporte[cite: 4, 6].
*   **Fluxo de Contingência**: Lógica para tratar falhas na entrega e iniciar processos de reentrega[cite: 5, 10].

### 🏃 Simulação de Evacuação Espacial
*   **Modelagem de Ambiente**: Tradução de um apartamento físico (com escadas, quartos trancados e áreas bloqueadas) em uma matriz de eventos lógicos.
*   **Gestão de Energia**: Implementação de uma camada de realidade onde o agente "cansa" e a simulação para se houver loops infinitos ou caminhos ineficientes.
*   **Navegação Crítica**: Uso de condicionais aninhadas para gerenciar pré-requisitos, como a abertura de trancas duplas para acessar a saída[cite: 14].

## 🧠 Reflexão Crítica (Aprendizados de ADS)
O processo de desenvolvimento permitiu percepções profundas sobre a engenharia de software[cite: 13, 14]:
*   **Tradução do Real**: Aprender a "quebrar a cabeça" para transformar obstáculos físicos em `if` e `else` funcionais[cite: 14].
*   **Filtragem de Essenciais**: O desafio de "limpar" o que não importa para manter o código focado no que trava ou libera o caminho[cite: 14].
*   **Lógica sob Pressão**: Perceber que programar é montar um plano de segurança onde cada linha de código serve para prever um erro e garantir a eficiência[cite: 14].
